
## Project Description

This issue addresses the main issue: for finding sports places near town. For example if an online user looks up online:
'basketballplatz konstanz' not many results are found and also in a bad quality. My app focuses on: 

-SEO optimized results ('tischtennis platte konstanz -> '5 Ergebnisse in 'konstanz' gefunden')
-qualitative results (with, pictures, details, ratings, comment section)
-community driven (a user can anonymously tick on a calendary, to show that he will be there at monday 5pm f.e.)

## Getting Started

First, run the development server (read first the prisma section):

```bash
pnpm run dev
```

```bash
next dev    -> starts the development server using Turbopack (default bundler)
next build  -> builds the application for production
next start  -> starts the production server
eslint      -> runs ESLint
```

The Prisma model is created/generated by 'db pull', which is basically the PostgreSQL schemas 
translated into prisma models (ORM). Run db pull if the PostgreSQL tables changed, 
then run prisma generate. Also run prisma generate after git clone.
```bash
pnpm prisma db pull
pnpm prisma generate
```


Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.
